# Workflow

## Minimal Maintenance Loop

1. Monitor public conference or proceedings sources.
2. Add or correct entries in the relevant manifest.
3. Regenerate conference indexes and sample outputs.
4. Run repository validation.
5. Update the changelog and maintenance notes.

## Typical Outputs

- conference paper tables
- topic maps
- weekly digests
- maintenance notes

## Design Principle

Prefer repeatable scripts and versioned manifests over manual one-off edits. The repository should make ongoing maintenance easier, not depend on hidden personal context.
