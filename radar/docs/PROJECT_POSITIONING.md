# Project Positioning

## Repository Role

Time-Series Research Radar is a public research-maintenance tool for people who need structured, reusable tracking of time-series papers across major venues.

## Public Narrative

Prefer wording like:

- "This project helps researchers reduce repetitive literature-tracking work."
- "This repository turns conference updates into reproducible public artifacts."
- "This project provides maintainable paper manifests, indexes, audits, and research digests."

Avoid centering the repository on a private personal workflow.

## Signals of Project Health

- ongoing conference updates and metadata corrections
- changelog and roadmap entries tied to real repository work
- issue handling or contribution review
- sample outputs that demonstrate the repository is usable today
- clear documentation for recurring maintenance workflows

## Principle

The repository should look and behave like a real, useful project under continuous maintenance, not a one-off data dump.
