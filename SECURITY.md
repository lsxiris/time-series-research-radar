# Security Policy

## Scope

This repository mainly stores public metadata, links, scripts, and generated documentation. Security issues can still matter when they affect:

- repository automation
- downloaded-file handling
- accidental credential exposure
- malicious links or unsafe generated content

## Reporting a Vulnerability

Please report vulnerabilities privately through GitHub security reporting when available. If the issue is not suitable for a public issue, do not open it in the regular issue tracker.

## Useful Report Details

- affected file or script path
- steps to reproduce
- expected impact
- whether the issue involves credentials, unsafe downloads, or supply-chain risk
